# Drive Store

ระบบดาวน์โหลด/อัปโหลดไฟล์ภายในองค์กรสำหรับ 13 สาขา
หน้าเว็บโฮสต์บน **GitHub Pages**, API หลังบ้านวิ่งบน **Google Apps Script (GAS)**,
ไฟล์ทั้งหมดเก็บใน **Google Drive**

---

## โครงสร้าง

```
index.html   ← เว็บหน้าเดียว (login + branch view + admin upload) ใช้กับ GitHub Pages
Code.gs      ← API ฝั่ง Google Apps Script (วางทั้งไฟล์เข้า Apps Script project)
```

## ขั้นตอนติดตั้ง

### 1. ติดตั้งฝั่ง Google Apps Script

1. เปิดสคริปต์
   https://script.google.com/d/1apw7_QPsqtnwA4EW9VibhtPVmQRFBSPHBg-kVOemhcaE1rFCTDPGLL3A/edit
2. ลบโค้ดเดิมใน `Code.gs` แล้ววางเนื้อหาจาก `Code.gs` ในรีโปนี้ทั้งหมด
3. กด **Save**
4. กด **Deploy → New deployment**
   - Type: **Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
5. คัดลอกลิงก์ `.../exec`
6. รันฟังก์ชัน `apiStores_` หนึ่งครั้งเพื่อขอ permission กับ Drive/Sheets

### 2. ติดตั้งฝั่ง Front-end (GitHub Pages)

1. แก้บรรทัด `API_URL` ใน `index.html` ให้เป็นลิงก์ `/exec` ที่ได้จากข้อ 1.5
2. Push ขึ้น GitHub
3. ที่ repo → **Settings → Pages → Source: Deploy from branch**, Folder: `/ (root)`
4. รอประมาณ 1 นาที จะได้ URL `https://<user>.github.io/<repo>/`

## วิธีใช้งาน

### สำหรับสาขา
- ใส่รหัสสาขา (เช่น `0730`, `3100`, ...) → เห็นเฉพาะไฟล์ของสาขาตัวเอง
- ไฟล์ที่ยังไม่ดาวน์โหลดจะมีป้าย **NEW** กระพริบ
- เมื่อกดดาวน์โหลด ป้าย NEW จะหายและการ์ดจะซีดลง

### สำหรับแอดมิน
- รหัสเริ่มต้น: `admin2026` (แก้ได้ใน `Code.gs` ที่ตัวแปร `ADMIN_CODE`)
- เลือกไฟล์ → ติ๊กสาขาที่จะส่งให้ → กด **อัปโหลด**
- ระบบสร้างโฟลเดอร์สาขาให้อัตโนมัติถ้ายังไม่มี
- ไฟล์ใหม่จะปรากฏที่หน้าจอสาขาเหล่านั้นพร้อมป้าย NEW ทันที

## หมายเหตุ

- ป้าย NEW ดูจาก Sheet `DownloadStatus` (สร้างอัตโนมัติเมื่อรันครั้งแรก)
- ขนาดไฟล์อัปโหลดสูงสุดต่อครั้งจำกัดที่ ~50 MB (ข้อจำกัดของ Apps Script)
- เปลี่ยนรหัสแอดมินที่ตัวแปร `ADMIN_CODE` ใน `Code.gs`
